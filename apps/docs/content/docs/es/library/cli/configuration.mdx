# Configuración

El CLI usa dos archivos de configuración.

## `~/.commet/auth.json`

Credenciales de autenticación globales.

**Creado por**: `commet login`

**Estructura**:
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "environment": "sandbox"
}
```

- `token`: Token de acceso
- `environment`: `"sandbox"` o `"production"`

---

## `.commet`

Configuración de organización del proyecto.

**Creado por**: `commet link`

**Estructura**:
```json
{
  "orgId": "org_A1b2C3d4E5f6G7h8I9j0K1l2",
  "orgName": "Acme Corporation",
  "environment": "sandbox"
}
```

---

## `.commet/types.d.ts`

Tipos TypeScript generados.

**Creado por**: `commet pull`

Contiene definiciones de tipos para tus event types, seat types y products.

---

## Entornos

Commet tiene dos entornos completamente aislados:

**Sandbox** (`sandbox.commet.co`): Desarrollo y pruebas

**Production** (`commet.co`): Facturación en vivo

Cambiar entornos:

```bash
commet logout
commet login  # Seleccionar entorno diferente
commet link
```

---

## Integración con Git

**Commitear `.commet`** si todos los miembros del equipo usan la misma organización.

**Ignorar `.commet`** si los desarrolladores usan organizaciones personales.

**Commitear `.commet/types.d.ts`** para tipos consistentes en el equipo.

**Ejemplo `.gitignore`**:
```bash
# Ignorar si se usan orgs personales
.commet
```
