# Gestión de Suscripciones

Las suscripciones son el corazón de Commet. Definen cómo y cuánto cobrar a tus clientes de manera recurrente, conectando productos, clientes y acuerdos en un flujo automatizado de facturación.

<Callout type="success">
La sección de Plataforma está diseñada para equipos de go-to-market, finanzas y operaciones. Si necesitas detalles de implementación, ve a la [Librería de Desarrolladores](/docs/library/quickstart).
</Callout>

## Ejemplo de inicio rápido

Así es como una empresa SaaS típica crea y gestiona suscripciones:

1. **Crear un cliente**: Registra la información del cliente con su correo de facturación
2. **Seleccionar productos**: Elige productos y variantes de precio del catálogo
3. **Iniciar suscripción**: Genera el checkout y envía el link de pago al cliente
4. **Facturación automática**: Una vez activa, Commet genera facturas recurrentes automáticamente

El motor de facturación maneja todo el ciclo: creación → checkout → activación → facturación recurrente.

## Por qué importan las suscripciones

- **Facturación recurrente automatizada**: Una vez activa, las suscripciones generan facturas automáticamente cada período sin intervención manual
- **Control total del ciclo de vida**: Gestiona estados, fechas de inicio, cancelaciones y renovaciones desde un solo lugar
- **Visibilidad completa**: Accede a historial de facturas, pagos y detalles del cliente en tiempo real
- **Flexibilidad operativa**: Crea suscripciones como borrador para revisión, programa fechas futuras o activa inmediatamente

## Ciclo de vida de una suscripción

Las suscripciones pasan por diferentes estados durante su vida útil:

| Estado | Descripción | Cuándo ocurre |
| --- | --- | --- |
`Borrador` |Suscripción creada pero no iniciada | Cuando creas una suscripción para revisión antes de enviar al cliente
| `Pendiente de pago` | Cliente recibió el checkout pero no completó el pago | Inmediatamente después de iniciar la suscripción |
| `Cancelada` (Checkout expirado) | El cliente no completó el pago en 24 horas | Automáticamente después de 24 horas sin pago |
| `Activa` | Cliente completó el pago y la suscripción está facturando | Después del primer pago exitoso |

## Cómo crear una suscripción

### Paso 1: Seleccionar o crear cliente

Navega a **Suscripciones** en la plataforma y haz clic en **Crear Suscripción**.

Si el cliente ya existe en tu sistema, búscalo por nombre o correo. Si es un cliente nuevo, puedes crearlo directamente desde el flujo de creación de suscripción.

<Callout type="warning">
**Importante**: El correo del cliente se usa como dirección de envío del link de pago y no puede modificarse en el checkout. Asegúrate de usar el correo correcto desde el inicio.
</Callout>

### Paso 2: Agregar productos

Selecciona los productos que quieres incluir en la suscripción:

- **Productos del catálogo**: Elige entre los productos que hayas creado previamente
- **Variantes de precio**: Selecciona la variante específica para este cliente (precio de lista o variante personalizada)
- **Ajustes personalizados**: Si el cliente requiere descuentos o condiciones especiales, puedes editar las condiciones del producto y generar nuevas variantes al momento

<Callout type="info">
**Variantes automáticas**: Si aplicas descuentos o modificas precios durante la creación, Commet genera automáticamente una variante de precio específica para este cliente.
</Callout>

### Paso 3:  Iniciar suscripción

Cuando la suscripción esté lista, haz clic en **Comenzar**. La suscripción se creará en estado "Pendiente de pago" y se generará automáticamente un link de checkout que puedes enviar al cliente.

## Detalles de la suscripción

Cada suscripción incluye información completa para gestionar el ciclo de facturación:

### Link de checkout

- **Ver link actual**: Accede al link de pago generado para el cliente
- **Regenerar link**: Si el checkout expiró o necesitas un nuevo link, puedes regenerarlo (solo disponible si la suscripción no está activa)
- **Compartir con cliente**: Copia el link para enviarlo por correo o integrarlo en tu flujo de onboarding

### Facturas generadas

- **Historial completo**: Visualiza todas las facturas generadas para esta suscripción
- **Facturación automática**: Cada período de facturación genera automáticamente una nueva factura
- **Detalles de pago**: Revisa estado de pago, fechas de vencimiento y métodos de pago

### Información del cliente

- **Datos de contacto**: Accede rápidamente a la información del cliente asociado
- **Historial de suscripciones**: Ve todas las suscripciones activas e históricas del cliente
- **Configuración de facturación**: Revisa moneda, términos de pago y dirección de facturación


## Mejores prácticas

- **Verifica correo del cliente**: Confirma el correo antes de crear el cliente, ya que no puede modificarse en el checkout
- **Aprovecha variantes de precio**: Usa variantes para condiciones permanentes del cliente en lugar de aplicar descuentos temporales
- **Monitorea estados pendientes**: Revisa regularmente suscripciones en "Pendiente de pago" para seguimiento activo
- **Regenera checkouts expirados**: Si un checkout expiró, regenera el link en lugar de crear una nueva suscripción

## Errores comunes a evitar

- **Correo incorrecto del cliente**: Verifica el correo antes de crear el cliente, ya que se usa para el checkout y no puede cambiarse
- **Crear múltiples suscripciones para el mismo cliente**: Si un checkout expiró, regenera el link en lugar de crear una nueva suscripción
- **Olvidar activar borradores**: Las suscripciones en borrador no generan facturas hasta ser activadas manualmente
- **No revisar variantes aplicadas**: Confirma que las variantes de precio aplicadas sean las correctas antes de iniciar la suscripción

## Integración mediante SDK

Todos los pasos descritos anteriormente pueden realizarse programáticamente a través de nuestro SDK. Esta es la forma recomendada para integrar Commet directamente en tu aplicación.

<Callout type="info">
**Para integraciones in-app**: Si estás construyendo una integración dentro de tu aplicación, usa el SDK en lugar de la interfaz de la plataforma. Esto te permite automatizar completamente el flujo de suscripciones y mantener todo sincronizado con tu sistema.
</Callout>

### Ventajas de usar el SDK

- **Automatización completa**: Crea y gestiona suscripciones directamente desde tu código sin intervención manual
- **Sincronización en tiempo real**: Mantén tus datos sincronizados con Commet automáticamente
- **Experiencia de usuario mejorada**: Integra el checkout directamente en tu aplicación sin redirigir a páginas externas
- **Escalabilidad**: Gestiona miles de suscripciones programáticamente sin limitaciones de la interfaz

### Flujo típico con SDK

1. **Crear cliente**: Usa `commet.customers.create()` cuando un usuario se registra en tu aplicación
2. **Crear suscripción**: Llama a `commet.subscriptions.create()` cuando el usuario selecciona un plan
3. **Obtener checkout**: Accede al link de checkout desde la respuesta de la suscripción
4. **Gestionar estados**: Monitorea cambios de estado mediante webhooks o consultas periódicas

Para ver ejemplos de código completos y la documentación completa de la API, visita la [Librería de Desarrolladores](/docs/library/features/subscriptions).
