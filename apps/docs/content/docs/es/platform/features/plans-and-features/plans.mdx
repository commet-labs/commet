# Planes

Los Planes son paquetes de facturación preconfigurados que combinan precios, características y intervalos de facturación. Cuando asignas un Plan a un Cliente, se crea automáticamente una Suscripción, haciendo que las suscripciones basadas en planes sean la base del modelo de facturación de Commet.

<Callout type="success">
La sección de Plataforma está diseñada para equipos de go-to-market, finanzas y operaciones. Si necesitas detalles de implementación, ve a la [Librería de Desarrolladores](/docs/library/quickstart).
</Callout>

## Ejemplo de inicio rápido

Así es como una empresa SaaS típica usa Planes:

1. **Crear Planes**: Define tus niveles de precios (Starter, Pro, Enterprise) con precios y características
2. **Asignar Plan a Cliente**: Selecciona un cliente y asigna un plan—la suscripción se crea automáticamente
3. **Facturación automática**: Una vez activa, Commet genera automáticamente facturas recurrentes en cada período de facturación
4. **Gestionar actualizaciones**: Cambia los planes de los clientes o actualiza las configuraciones de planes según sea necesario

El motor de facturación maneja todo el ciclo: Creación de Plan → Asignación de Plan → Creación de Suscripción → Checkout → Activación → Facturación recurrente.

## Por qué importan los planes

- **Consistencia**: Estandariza los precios en todos los clientes que usan el mismo plan
- **Suscripciones automáticas**: Asignar un plan a un cliente crea automáticamente su suscripción
- **Actualizaciones fáciles**: Modifica las características o precios del plan y todas las futuras suscripciones usan el plan actualizado
- **Públicos o privados**: Controla qué planes aparecen en tu página de precios frente a planes solo internos
- **Escalabilidad**: Crea una vez, asigna a muchos clientes sin configuración manual

## ¿Qué son los planes?

Los Planes son plantillas que definen todo lo que un cliente obtiene cuando se suscribe:

| Componente | Descripción | Ejemplo |
| --- | --- | --- |
| **Nombre y Descripción** | Información de visualización para clientes y tu equipo | "Plan Pro" - "Mejor para equipos en crecimiento" |
| **Precios** | Múltiples opciones de precios por intervalo de facturación | $99/mes, $990/año (ahorra 17%) |
| **Características** | Lo que está incluido en el plan | Llamadas API, almacenamiento, asientos, flags de características |
| **Días de Prueba** | Período de prueba gratuito opcional | 14 días de prueba gratuita |
| **Visibilidad Pública** | Si el plan aparece en la página de precios | Público o privado |

Piensa en los Planes como paquetes de productos. Una vez que creas un Plan, puedes asignarlo a múltiples clientes, y cada asignación crea automáticamente una Suscripción vinculada a ese plan.

## Cómo crear un plan

<Steps>
<Step>

### Navegar a Planes

Ve a **Planes** en la navegación de la plataforma y haz clic en **Crear Plan**.

</Step>
<Step>

### Configurar detalles del plan

Configura la información básica del plan:

- **Nombre del plan**: Elige un nombre claro orientado al cliente (ej: "Starter", "Pro", "Enterprise")
- **Descripción**: Añade una breve descripción que explique la propuesta de valor del plan
- **Público o privado**: 
  - **Público**: El plan aparece en tu página de precios y puede ser seleccionado por los clientes
  - **Privado**: El plan solo es visible internamente (útil para acuerdos empresariales personalizados)

</Step>
<Step>

### Establecer precios

Añade precios para diferentes intervalos de facturación:

- **Mensual**: Facturación recurrente mensual
- **Trimestral**: Facturación recurrente cada 3 meses (a menudo con descuento)
- **Anual**: Facturación recurrente anual (a menudo con el mejor descuento)

Para cada intervalo de facturación:
- Ingresa el monto del precio
- Establece uno como predeterminado (se muestra primero cuando los clientes ven los planes)

<Callout type="info">
**Estrategia de precios**: Muchas empresas ofrecen descuentos por compromisos más largos. Por ejemplo: $99/mes, $270/trimestre (ahorra 9%), $990/año (ahorra 17%). Esto fomenta suscripciones anuales mientras mantiene la flexibilidad.
</Callout>

</Step>
<Step>

### Añadir características

Configura lo que está incluido en el plan añadiendo características. Las características definen lo que los clientes pueden acceder:

- **Características booleanas**: Capacidades de activar/desactivar (ej: SSO habilitado, marca personalizada)
- **Características medidas**: Límites basados en uso (ej: 10,000 llamadas API incluidas, luego $0.01 por llamada)
- **Características de asientos**: Licencias por usuario (ej: 5 asientos de editor incluidos, $10 por asiento adicional)

Puedes añadir múltiples características de cada tipo a un plan. Para información detallada sobre cómo configurar características, consulta la [documentación de Características](./features).

</Step>
<Step>

### Configurar período de prueba (opcional)

Si quieres ofrecer una prueba gratuita:

- Establece el número de días de prueba (ej: 14 días)
- Las suscripciones creadas con este plan comenzarán en estado "En Prueba"
- Los clientes no serán cobrados hasta que termine el período de prueba

</Step>
<Step>

### Guardar y publicar

Una vez configurado, guarda tu plan. Si está marcado como público, aparecerá en tu página de precios. Si es privado, solo tu equipo puede asignarlo a clientes.

</Step>
</Steps>

## Estructura del plan

### Información básica

| Campo | Descripción | Ejemplo |
| --- | --- | --- |
| **Nombre** | Nombre de visualización para el plan | "Plan Pro" |
| **Descripción** | Breve explicación del valor del plan | "Mejor para equipos en crecimiento" |
| **Código** | Identificador interno (generado automáticamente) | "pro" |
| **Público** | Si el plan aparece en la página de precios | Sí/No |
| **Orden de Clasificación** | Orden de visualización en la página de precios | 1, 2, 3... |

### Precios

Cada plan puede tener múltiples precios para diferentes intervalos de facturación:

| Intervalo de Facturación | Descripción | Uso Típico |
| --- | --- | --- |
| **Mensual** | Facturación recurrente mensual | Opción predeterminada, máxima flexibilidad |
| **Trimestral** | Facturación recurrente cada 3 meses | Pequeño descuento, compromiso trimestral |
| **Anual** | Facturación recurrente anual | Mejor descuento, compromiso anual |

Cuando se asigna un plan a un cliente, selecciona qué intervalo de facturación desea. La suscripción usará este intervalo para todos los futuros ciclos de facturación.

### Configuración de prueba

| Configuración | Descripción | Ejemplo |
| --- | --- | --- |
| **Días de Prueba** | Número de días gratuitos antes de que comience la facturación | 0 (sin prueba) a 365 |
| **Comportamiento de Prueba** | Cómo la prueba afecta el estado de la suscripción | La suscripción comienza en estado "En Prueba" |

Si un plan tiene días de prueba configurados, todas las suscripciones creadas con ese plan comenzarán automáticamente en modo de prueba.

## Cómo los planes se conectan a las suscripciones

Cuando asignas un Plan a un Cliente:

1. **La Suscripción se crea automáticamente** — No se necesita creación manual de suscripción
2. **La configuración del Plan se copia** — La suscripción hereda todas las configuraciones del plan (precios, características, prueba)
3. **Se establece el intervalo de facturación** — El cliente selecciona mensual, trimestral o anual en la asignación
4. **Se determina el estado** — La suscripción comienza en el estado apropiado:
   - **En Prueba**: Si el plan tiene días de prueba
   - **Pago Pendiente**: Si se requiere pago
   - **Activa**: Si no se necesita pago o prueba

Una vez asignada, la suscripción es independiente—los cambios en el plan no afectan las suscripciones existentes, solo las nuevas asignaciones.
